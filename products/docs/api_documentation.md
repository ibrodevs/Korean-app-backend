# Документация API для приложения Products

Данная документация описывает функционал предоставленных эндпоинтов приложения `products`. Все эндпоинты поддерживают мультиязычность через параметр `lang` (по умолчанию `ru`).

## Базовый формат ответов
- В качестве механизмов пагинации для списков применяется `CursorPagination` (по умолчанию размер страницы 40 элементов, сортировка по убыванию `created_at`).

---

## 1. Список товаров 
**GET** `/api/v1/products/`
(Базовый URL зависит от конфигурации `urls.py` проекта, может быть просто `/products/`)

Возвращает курсорно-пагинируемый список активных (где `is_active=True`) товаров со всеми необходимыми данными (категория, бренд, локализованные переводы, изображения).

### Поддерживаемые фильтры (Query Parameters)
- `is_active` (boolean) — фильтр по статусу активности (впрочем, код по умолчанию отсеивает неактивные товары).
- `category` (string) — фильтр по slug категории (например: `?category=electronics`).
- `brand` (string) — фильтр по slug бренда (например: `?brand=samsung`).
- `price[min]` (number) — фильтрация по минимальной цене товара (например: `?price[min]=100`).
- `price[max]` (number) — фильтрация по максимальной цене товара.
- **Динамические EAV фильтры**: `attr_{slug}` (string / arrays) — позволяют фильтровать по динамическим атрибутам товаров.
  - *Пример:* `?attr_ram=8` или `?attr_color=red&attr_color=black`.
  - Поддерживает числа, булевы значения и строковые совпадения для текста/цветов.
- `search` (string) — текстовый поиск по названию товара (`translations__name`) и описанию (`translations__description`).
- `lang` (string) — язык возвращаемых переводов (по умолчанию `ru`, возможны: `en`, `kg` и т.д.).

---

## 2. Детальная карточка товара
**GET** `/api/v1/products/<slug:slug>/`

Возвращает детальную карточку конкретного товара по его `slug`. 

### Поддерживаемые параметры
- `lang` (string) — язык возвращаемых переводов (по умолчанию `ru`).

### Функционал:
В ответ возвращается сам товар, его категория, бренд и все связанные сущности:
- Изображения (галерея товара).
- Варианты товаров (variants) со своими изображениями.
- Одиночные и множественные атрибуты для каждого варианта с переведенными значениями.

---

## 3. Фасетный поиск по каталогу
**GET** `/api/v1/catalog-search/`

Используется для построения умных страниц каталога с возможностью динамической фильтрации (фасеты для сайдбара). Наследует функциональность из списка товаров.

### Поддерживаемые фильтры и параметры (включая все параметры из п. 1):
- `lang` (string) — язык возвращаемых ответов.

### Функционал:
Кроме результатов выборки товаров (массив `results`), API также отдает ключ `facets` для построения пользовательского интерфейса:
- `brands`: доступные для текущей выборки бренды товаров с подсчетом совпадений (count).
- `price`: минимальная и максимальная цена найденных товаров.
- `attributes`: динамически собранный список значений атрибутов для текущих найденных товаров с подсчетом доступного количества.

---

## 4. Дерево категорий
**GET** `/api/v1/categories/`

Возвращает иерархическое дерево корневых категорий (не имеющих родителей) вместе со всеми вложенными в них дочерними подкатегориями `children`.

### Поддерживаемые параметры
- `lang` (string) — язык возвращаемых переводов (по умолчанию `ru`).

### Функционал:
Используется для полного построения навигационного меню или каталога категорий на клиентской части. Данные возвращаются со всеми переводами по одному запросу.

---

## 5. Список брендов
**GET** `/api/v1/brands/`

Возвращает плоский список всех доступных брендов в системе.

### Поддерживаемые параметры
- `lang` (string) — язык возвращаемых локализованных названий (по умолчанию `ru`).

### Функционал:
Предоставляет список брендов для построения селекта, страницы брендов или фильтра на стороне клиента.
